🧪 Laboratory Checkpoints (Lab 1 - Lab 5)

The foundational training series where I developed the baseline components for my application's 7-screen architecture (Activity, Detail, Navigation, Profile, Review, Saved, and Weather):

  • 📦 Lab 1: Environment Orchestration - Setting up the Android Studio IDE, local Android SDK configs, and compiling the initial Gradle build files.
  • 📦 Lab 2: Declarative UI Layouts - Implementing custom modifiers, layout scaffolding, and handling structural elements within Jetpack Compose.
  • 📦 Lab 3: Navigation Graph & UI State - Wiring the core Jetpack NavHost engine to control deep-link parameters and route parameters between view states.
  • 📦 Lab 4: Async Network Interfacing - Utilizing a Retrofit client to handle remote HTTP calls, querying remote endpoints, and structural JSON stream parsing.
  • 📦 Lab 5: Local Room SQL Storage - Establishing local edge database persistence, configuring the structural entities, and writing native Data Access Objects (DAO).

🏆 Project 2: Advanced Trail Tracker App (Final VSR Deployment)

STRICT 3-MINUTE VSR DEMO: The ultimate implementation phase executing the fully wired, sensor-aware 7-screen eco-hiking system. The demo illustrates clicking the dashboard discovery button to pull dynamic datasets from MockAPI, selecting the UKM Loop trail route, and running an active tracking session in the NavigationScreen.

It showcases native hardware Android GPS Sensor callback tracking, live weather telemetry updates parsed through the Open-Meteo REST service in the WeatherScreen, and automated data syncing architectures (saving local offline caches via Room DB and syncing up to the Firebase Cloud database) along with a walk-through of the underlying Kotlin source files.
