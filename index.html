Project 2: Sensor-Aware Full-Stack Architecture (Lab 5 Extension - Main Piece)

Problem Statement: Outdoor trail trackers break functionality in deep forest terrains due to lack of local persistence fallback, remote sync systems, and real-time sensor updates.

Description: A strict 3-minute execution demo illustrating a fully wired, sensor-aware 7-screen ecosystem. Features active trail fetching from MockAPI (e.g., UKM Loop), real-time Android GPS tracking (NavigationScreen), Open-Meteo REST parameters (WeatherScreen), and full multi-tier caching (Room SQLite locally + Firebase database cloud sync).

Lessons Learned: Managed advanced asynchronously executed data integration using native Room databases coupled with live Firebase cloud pushes, live Open-Meteo REST parameters, and structural Android hardware GPS callback channels.

4. Learning Journey Reflection

🚀 Evolution (From Lab 1 to Project 2): My engineering confidence shifted fundamentally. I began in Lab 1 by creating hardcoded UI assets. By the time I reached Project 2, I was independently architecting reactive Android software capable of mapping real-time geospatial configurations and caching multi-tier datasets.

⚠️ Technical Challenges Overcome: Handling thread-safety concurrency issues while mapping dynamic hardware location updates onto localized database streams represented a critical learning roadblock. I mitigated this by wrapping continuous GPS emission streams inside localized async background Coroutines.

🌳 SDG Structural Impact: Developing this eco-hiking solution bridges digital infrastructure with active environmental safety frameworks (SDG 15). Providing data-driven mapping variables keeps community hikers informed, reducing accidents and promoting sustainable trail habits.
